# PHPThis 0.1.0-alpha.1

Status: unpublished draft; version identity approved; release candidate not yet frozen.

- Composer version: `0.1.0-alpha.1`
- Framework tag: `v0.1.0-alpha.1`
- Skeleton tag: `v0.1.0-alpha.1`
- Release date: not set
- Candidate commit: not set

These notes describe the intended bounded Alpha 1 claim. They are not an announcement that either package or the public installation path is available.

## Purpose

Alpha 1 is an experimental foundation for AI-first PHP authoring with human accountability. It keeps request handling, routing, dependencies, SQL, state, and resource limits visible in ordinary typed PHP, then makes the accepted subset executable through one installed check.

This prerelease is for evaluation and feedback. It is not production-ready and makes no backward-compatibility promise across prereleases.

## Included surface

- Consumer Contract version 3 and Strict Profile version 2, including permanent diagnostics `PHT001` through `PHT006`.
- The installed `phpthis check` command, framework-owned maximum-level PHPStan configuration, repository guardrails, and mandatory application-owned automated behavior-test stage.
- Manual constructor composition, immutable request and response values, bounded PHP runtime request ingestion, exact-class error mapping, validated response cookies, and explicit response emission.
- Indexed literal routes and the one bounded trailing `{name:positive-int}` route form, with immutable typed parameter delivery.
- A thin PDO transport with finite compile-time-constant SQL, unique named bindings, explicit transactions, query budgets, and bounded redacted query traces.
- Optional lazy native-file session transport behind an explicit lifecycle and typed application-owned state boundaries.
- An independently packageable health-only application skeleton with project-owned AI context and a complete profile-plus-behavior-test gate.

## Explicit exclusions and limitations

Alpha 1 does not provide or promise:

- production readiness, a support or security-response SLA, or backward compatibility between prereleases;
- a generic CRUD runtime or complete Create identity and conflict behavior, Update, or Delete;
- authentication, authorization, tenant isolation, CSRF policy, or application session-security policy;
- migrations, queues, templating, uploads, streaming, workers, a cache mechanism, or a production deployment topology;
- SQL dialect portability merely because the PDO transport is certified against SQLite, MySQL, and PostgreSQL.

Application SQL, schema policy, runtime database authority, identity, conflicts, security, cache policy, and operational topology remain explicit application-owned decisions.

## Publication and installation status

The public `composer create-project --stability=alpha phpthis/skeleton` path is intentionally unavailable until both packages are indexed from their approved tags and the unchanged command passes the complete public-distribution proof in `RELEASING.md`. Do not publish an installation command or announce Alpha 1 before that proof succeeds.

Before publication, the final candidate must pass the complete local gate, both GitHub CI jobs, exact framework archive inventory comparison, skeleton export and locked dependency proof, Packagist-preferred distribution checks for both packages, and the clean public installation and health checks. Record the final candidate commit, release date, artifact references, and evidence before replacing this draft status.

## Upgrade notes

This is the intended first public prerelease, so there is no earlier released PHPThis version to upgrade from. Future prereleases may contain breaking changes; each must keep its contract, diagnostics, decisions, and upgrade impact explicit.
